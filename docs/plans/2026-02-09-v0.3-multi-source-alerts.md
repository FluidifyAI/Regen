# v0.3 Multi-Source Alerts, Grouping & Routing ŌĆö Implementation Plan

## Context

v0.3 transforms OpenIncident from a **Prometheus-only** alert receiver into a **multi-source alert platform**. This is the release where OpenIncident becomes genuinely useful for teams running heterogeneous monitoring stacks.

**Current State (v0.2):**
- Single webhook: `POST /api/v1/webhooks/prometheus`
- `AlertService.ProcessAlertmanagerPayload()` tightly coupled to Alertmanager format
- Basic deduplication: `(source, external_id)` handles exact duplicates from Prometheus
- No grouping: each alert creates its own incident
- No routing: all incidents go to auto-named Slack channels

**Target State (v0.3):**
- Four webhooks: Prometheus, Grafana, CloudWatch, Generic
- `WebhookProvider` interface normalizes any source into `NormalizedAlert`
- Grouping rules combine related alerts into single incidents
- Routing rules direct alerts to correct channels/teams
- Full pipeline: **Normalize ŌåÆ Deduplicate ŌåÆ Group ŌåÆ Route ŌåÆ Create Incident**

**User Stories:**
1. As a DevOps engineer using Grafana, I want to point Grafana alerts at OpenIncident so I have one place for all incidents.
2. As an AWS user, I want CloudWatch alarms to auto-create incidents without building a Lambda bridge.
3. As a team lead, I want related alerts (same service, same time window) grouped into one incident, not five.
4. As a platform engineer, I want to route database alerts to #db-oncall and API alerts to #api-oncall automatically.

## Architectural Decisions

### 1. WebhookProvider Adapter Pattern (Strategy Pattern)

**Decision:** Each monitoring source implements a `WebhookProvider` interface that normalizes source-specific payloads into `[]NormalizedAlert`.

```go
type WebhookProvider interface {
    Source() string
    ValidatePayload(body []byte, headers http.Header) error
    ParsePayload(body []byte) ([]NormalizedAlert, error)
}
```

**Why this pattern:**
- Mirrors the existing `ChatService` interface (proven pattern in this codebase)
- Each provider is independently testable with fixtures
- Adding a new source (Datadog, PagerDuty import, etc.) requires only implementing the interface
- Core AlertService remains unchanged ŌĆö it only sees NormalizedAlerts
- Avoids a monolithic switch statement in the handler

**Trade-off:** More files (one per provider) vs. concentrated logic. The modularity wins for maintainability.

### 2. NormalizedAlert as Internal Interchange Format

**Decision:** All webhook providers output `NormalizedAlert`, which is the only type `AlertService.ProcessNormalizedAlerts()` accepts.

```go
type NormalizedAlert struct {
    ExternalID  string              // Source-specific dedup key
    Source      string              // "prometheus", "grafana", "cloudwatch", "generic"
    Status      string              // "firing" | "resolved"
    Severity    string              // "critical" | "warning" | "info"
    Title       string              // Human-readable alert title
    Description string              // Detailed description
    Labels      map[string]string   // Normalized key-value labels
    Annotations map[string]string   // Additional metadata
    RawPayload  json.RawMessage     // Complete original payload
    StartedAt   time.Time
    EndedAt     *time.Time
}
```

**Why a separate type (not reusing Alert model):**
- `Alert` model has database concerns (ID, timestamps, GORM tags)
- `NormalizedAlert` is a pure data transfer object ŌĆö no persistence logic
- Keeps the boundary clean between "what we received" and "what we stored"

### 3. Pipeline Architecture

**Decision:** Alert processing follows a linear pipeline with optional stages:

```
Webhook Request
    Ōåō
[1. Provider.ValidatePayload()] ŌåÆ 401 if invalid signature
    Ōåō
[2. Provider.ParsePayload()]    ŌåÆ 400 if malformed
    Ōåō
[3. Deduplicate]                ŌåÆ Update existing alert if duplicate
    Ōåō
[4. GroupingEngine.Evaluate()]  ŌåÆ Link to existing incident if grouped
    Ōåō
[5. RoutingEngine.Evaluate()]   ŌåÆ Determine channel, severity overrides
    Ōåō
[6. Create/Update Incident]     ŌåÆ Apply routing decisions
    Ōåō
[7. Async: Slack notifications] ŌåÆ Channel creation, message posting
    Ōåō
HTTP 200 Response
```

**Why linear pipeline:**
- Each stage has clear input/output
- Stages can be disabled independently (no grouping rules = skip step 4)
- Debugging: logs show exactly which stage produced which result
- Performance: no unnecessary branching or parallel evaluation

### 4. Grouping: Database Advisory Locks for Concurrency

**Decision:** Use PostgreSQL advisory locks on the grouping key to prevent race conditions when concurrent webhooks try to create the same incident group.

```go
// Pseudo-code
groupKey := deriveGroupKey(alert, rule) // e.g., "service=api,env=prod"
lockID := hash(groupKey)
db.Exec("SELECT pg_advisory_xact_lock($1)", lockID)
// Now safe to check for existing incident and create if needed
```

**Why advisory locks over application-level mutexes:**
- Works across multiple OpenIncident instances (horizontal scaling)
- Automatic release on transaction commit/rollback
- No deadlock risk with transaction-scoped locks
- PostgreSQL handles all the hard concurrency problems

### 5. Routing: First-Match-Wins with Priority

**Decision:** Routing rules are evaluated in priority order. First matching rule determines the routing decision. Unmatched alerts use default behavior.

**Why first-match-wins:**
- Predictable behavior ŌĆö users know exactly which rule will fire
- Matches mental model from firewall rules, Nginx location blocks, Alertmanager routing
- Priority field makes ordering explicit (not insertion-order dependent)
- Default catch-all rule is simply the lowest-priority rule

### 6. SNS Subscription Confirmation (CloudWatch)

**Decision:** Auto-confirm SNS subscriptions in the webhook handler rather than requiring manual confirmation.

**Why:**
- CloudWatch webhook setup should be zero-friction
- Manual confirmation would require users to copy/paste a URL ŌĆö bad UX
- Security: still validate SNS message signature before confirming
- One-time operation per subscription

#### OI-090: WebhookProvider Interface + NormalizedAlert

**Create:** `backend/internal/models/webhooks/provider.go`

```go
package webhooks

import (
    "encoding/json"
    "net/http"
    "time"
)

// WebhookProvider normalizes monitoring source payloads into NormalizedAlerts.
// Each monitoring source (Prometheus, Grafana, CloudWatch, etc.) implements this
// interface to handle its specific payload format, authentication, and field mapping.
type WebhookProvider interface {
    // Source returns the source identifier (e.g., "prometheus", "grafana").
    // Used as the alert.source field for storage and deduplication.
    Source() string

    // ValidatePayload checks authentication (signatures, tokens) before parsing.
    // Returns nil if valid or no auth configured.
    ValidatePayload(body []byte, headers http.Header) error

    // ParsePayload converts a raw webhook body into normalized alerts.
    // Each source maps its specific fields to the canonical NormalizedAlert format.
    ParsePayload(body []byte) ([]NormalizedAlert, error)
}

// NormalizedAlert is the canonical internal representation of an alert.
// All webhook providers produce NormalizedAlerts; AlertService only consumes this type.
type NormalizedAlert struct {
    ExternalID  string            `json:"external_id"`
    Source      string            `json:"source"`
    Status      string            `json:"status"`      // "firing" | "resolved"
    Severity    string            `json:"severity"`     // "critical" | "warning" | "info"
    Title       string            `json:"title"`
    Description string            `json:"description"`
    Labels      map[string]string `json:"labels"`
    Annotations map[string]string `json:"annotations"`
    RawPayload  json.RawMessage   `json:"raw_payload"`
    StartedAt   time.Time         `json:"started_at"`
    EndedAt     *time.Time        `json:"ended_at,omitempty"`
}
```

#### OI-091: Refactor AlertService

**Modify:** `backend/internal/services/alert_service.go`

Add `ProcessNormalizedAlerts()` method. Extract the core logic from `ProcessAlertmanagerPayload()`:

```go
// New method ŌĆö source-agnostic
func (s *alertService) ProcessNormalizedAlerts(alerts []webhooks.NormalizedAlert) (*ProcessingResult, error) {
    result := &ProcessingResult{}
    for _, normalized := range alerts {
        alert := s.normalizedToModel(normalized)
        created, err := s.createOrUpdateAlert(alert)
        if err != nil {
            return nil, err
        }
        if created {
            result.Created++
            if s.shouldCreateIncident(alert.Severity) {
                incident, err := s.incidentSvc.CreateIncidentFromAlert(alert)
                if err == nil {
                    result.IncidentsCreated++
                }
            }
        } else {
            result.Updated++
        }
    }
    return result, nil
}

// Existing method ŌĆö delegates to new method
func (s *alertService) ProcessAlertmanagerPayload(payload *webhooks.AlertmanagerPayload) (*ProcessingResult, error) {
    provider := &webhooks.PrometheusProvider{}
    normalized, err := provider.ParsePayload(payloadBytes)
    if err != nil {
        return nil, err
    }
    return s.ProcessNormalizedAlerts(normalized)
}
```

type PrometheusProvider struct{}

func (p *PrometheusProvider) Source() string { return "prometheus" }

func (p *PrometheusProvider) ValidatePayload(body []byte, headers http.Header) error {
    return nil // Prometheus uses webhook URL secrecy
}

func (p *PrometheusProvider) ParsePayload(body []byte) ([]NormalizedAlert, error) {
    var payload AlertmanagerPayload
    if err := json.Unmarshal(body, &payload); err != nil {
        return nil, fmt.Errorf("invalid alertmanager payload: %w", err)
    }

    alerts := make([]NormalizedAlert, 0, len(payload.Alerts))
    for _, a := range payload.Alerts {
        alerts = append(alerts, NormalizedAlert{
            ExternalID:  a.Fingerprint,
            Source:      "prometheus",
            Status:      mapPrometheusStatus(a.Status),
            Severity:    extractSeverity(a.Labels),
            Title:       a.Labels["alertname"],
            Description: firstNonEmpty(a.Annotations["summary"], a.Annotations["description"]),
            Labels:      a.Labels,
            Annotations: a.Annotations,
            RawPayload:  body,
            StartedAt:   a.StartsAt,
            EndedAt:     parseEndedAt(a.EndsAt),
        })
    }
    return alerts, nil
}

#### OI-096: Unified Webhook Handler

**Create:** `backend/internal/api/handlers/webhook_handler.go`

```go
type WebhookHandler struct {
    provider     webhooks.WebhookProvider
    alertService services.AlertService
}

func NewWebhookHandler(provider webhooks.WebhookProvider, alertService services.AlertService) *WebhookHandler {
    return &WebhookHandler{provider: provider, alertService: alertService}
}

func (h *WebhookHandler) Handle(c *gin.Context) {
    body, err := io.ReadAll(io.LimitReader(c.Request.Body, maxPayloadSize))
    if err != nil {
        c.JSON(400, gin.H{"error": "failed to read request body"})
        return
    }

    if err := h.provider.ValidatePayload(body, c.Request.Header); err != nil {
        c.JSON(401, gin.H{"error": "authentication failed", "detail": err.Error()})
        return
    }

    alerts, err := h.provider.ParsePayload(body)
    if err != nil {
        c.JSON(400, gin.H{"error": "invalid payload", "detail": err.Error()})
        return
    }

    result, err := h.alertService.ProcessNormalizedAlerts(alerts)
    if err != nil {
        c.JSON(500, gin.H{"error": "processing failed"})
        return
    }

    c.JSON(200, gin.H{
        "source":            h.provider.Source(),
        "received":          len(alerts),
        "created":           result.Created,
        "updated":           result.Updated,
        "incidents_created": result.IncidentsCreated,
    })
}
```

### Key Design: Grouping Rules Engine

```go
type GroupingEngine struct {
    ruleRepo GroupingRuleRepository
    cache    *sync.Map // rule cache with TTL
}

func (g *GroupingEngine) EvaluateAlert(alert NormalizedAlert) (*GroupingDecision, error) {
    rules := g.getActiveRules() // cached, refreshed every 30s

    for _, rule := range rules {
        if g.matchesRule(alert, rule) {
            groupKey := g.deriveGroupKey(alert, rule)
            existingIncident := g.findOpenIncidentForGroup(groupKey, rule.TimeWindow)
            if existingIncident != nil {
                return &GroupingDecision{
                    Action:     GroupActionLinkToExisting,
                    IncidentID: existingIncident.ID,
                    RuleName:   rule.Name,
                    GroupKey:   groupKey,
                }, nil
            }
            // No existing incident for this group ŌĆö create new
            return &GroupingDecision{
                Action:   GroupActionCreateNew,
                RuleName: rule.Name,
                GroupKey: groupKey,
            }, nil
        }
    }

    // No rule matched ŌĆö use default behavior
    return &GroupingDecision{Action: GroupActionDefault}, nil
}
```

## Risk Mitigation

| Risk | Impact | Mitigation |
|------|--------|------------|
| Grafana payload format changes between versions | Alerts fail to parse | Support both legacy and Unified Alerting formats; version detection from payload structure |
| SNS signature verification complexity | CloudWatch webhook insecure or too complex | Use AWS SDK's message verification; fall back to permissive mode with warning log |
| Grouping race conditions under load | Duplicate incidents for same group | PostgreSQL advisory locks; integration tests with concurrent goroutines |
| Routing rule misconfiguration suppresses all alerts | Incidents not created | Default catch-all rule cannot be deleted; test endpoint for dry-run validation |
| Performance degradation with many rules | Slow webhook processing | In-memory rule cache with 30s TTL; compiled regex cache; <10ms target |
| Breaking existing Prometheus integration | Regression for existing users | Prometheus refactor is behavior-preserving; existing test suite is the safety net |
